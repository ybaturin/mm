# Mechanical PEAD Edge Study — Design Spec

**Дата:** 2026-06-16
**Статус:** утверждён к реализации
**Автор:** Iurii Baturin

## 1. Цель

Определить, есть ли у **механического PEAD** (post-earnings-announcement drift, без LLM)
**торгуемый, переживающий реальные издержки, стабильный out-of-sample edge** для малого
счёта — до единого реального доллара.

Это исследовательский стади, не торговая система. Вывод — гейт: строить ли торговлю на
этой аномалии.

### Контекст (что привело сюда)

Измеритель LLM-edge (`docs/superpowers/specs/2026-06-14-llm-edge-measurer-design.md`)
дал чистый результат на двух пред-зарегистрированных выборках:

| n≈105, 5 дней | large-cap | mid/small-cap |
|---|---|---|
| LLM IC | −0.074 | +0.102 |
| **Тупой PEAD IC** | **+0.056** | **+0.147** |
| Long-short после изд. | −4.4% | +3.1% |
| t-stat | −0.52 | +2.20 |

Два вывода: (1) **глубокое чтение LLM не добавляет edge** — однострочная формула
EPS-surprise обыгрывала Claude на обеих выборках; гипотеза «LLM читает тон лучше рынка»
опровергнута. (2) **Механический PEAD подаёт признаки жизни на small-cap** (IC +0.147,
long-short +3.1%, t=2.2) — единственная нить с эмпирической поддержкой из наших данных.

Этот стади **выкидывает LLM** и строго проверяет именно механическую нить.

## 2. Принципы

- **Измерить, потом торговать.** Ни доллара, пока edge не доказан на held-out.
- **Без LLM → бесплатно и быстро.** Только EPS (Alpha Vantage `EARNINGS`) + цены
  (yfinance). Ноль токенов; прогон — минуты, не часы (нет последовательных LLM-вызовов).
- **Реалистичные издержки решают.** Small-cap спред+импакт >> плоские 10бп; без честной
  cost-модели любой «edge» фиктивен.
- **Защита от переобучения важнее красивой цифры.** Мы делаем развёртку параметров →
  multiple testing. Без train/test split и пред-регистрации найденный edge = артефакт.
- **Современный PEAD подъеден.** Базовое ожидание — слабый сигнал, возможно съедаемый
  издержками. Стади дешёвое — стоит одного строгого замера.
- **Переиспользуем** `edge/`: данные, метрики, point-in-time дисциплину.

## 3. Сигнал — SUE

Standardized Unexpected Earnings, литературный стандарт (сильнее, чем sign-only из
измерителя):

```
surprise = reportedEPS − estimatedEPS
SUE      = surprise / normalizer
```

Два варианта нормализации (обе считаются, сравниваются):
- **по цене:** `surprise / price_at_decision` — устойчиво, не требует истории.
- **по σ сюрпризов:** `surprise / std(прошлые 8 surprise того же тикера)` — классический
  SUE; требует ≥4-8 прошлых отчётов (из `EARNINGS`, point-in-time).

Знак SUE = направление ставки; |SUE| = сила (для квинтильной сортировки и порога).

## 4. Развёртки (что варьируем и меряем)

Сетка измеряется **только на train-половине** (см. §7):

- **Cap-tier:** large / mid / small (`config/edge_universe_full.toml` +
  `edge_universe_smid.toml` + явный small-список). Гипотеза: edge концентрируется в small.
- **Горизонт:** 1 / 5 / 10 / 20 / 60 торговых дней. Классический PEAD дрейфит до **60
  дней** — мы смотрели только 5; дрейф может быть глубже и устойчивее на длинном конце.
- **Порог сюрприза:** торговать все vs только |SUE| выше порога (топ-квинтиль).
- **Нормализация:** по цене vs по σ.

## 5. Реалистичные издержки (make-or-break)

Round-trip издержки по тиру (спред + проскальзывание + комиссия), вместо плоских 10бп:

| Тир | Round-trip (bps) | Обоснование |
|---|---|---|
| large | ~5 | узкие спреды, глубокая ликвидность |
| mid | ~20 | умеренные спреды |
| small | ~60 | широкие спреды + импакт на малой ликвидности |

Числа — стартовые, конфигурируемые; для лонг-шорт пары издержки удваиваются (две ноги).
Каждая метрика доходности считается **после** издержек тира. Это, по гипотезе, и есть
то, что убьёт small-cap +3.1%.

## 6. Портфель-симуляция и метрики

- **Конструкция:** на каждую дату отчётов — лонг топ-квинтиль SUE, шорт дно-квинтиль
  (или лонг-онли вариант для счёта без шортов). Держим горизонт, выходим. Перекрытие
  позиций допускается (несколько отчётов в окне).
- **Метрики:** market-adjusted (минус SPY), net издержек тира — IC, long-short спред,
  hit rate, t-stat, **equity curve → annualized Sharpe, max drawdown** (переиспуем
  `analysis/track_record.py`).
- **Бенчмарки:** монетка + buy-and-hold SPY за тот же период.

## 7. Защита от переобучения (ядро стади)

Развёртка §4 = перебор → на одной выборке что-то «выстрелит» случайно. Дисциплина:

1. **Train/test split по времени.** Post-cutoff окно делится: ранняя половина — train
   (поиск конфигурации), поздняя — **held-out test** (трогаем один раз).
2. **Пред-регистрация.** На train выбираем ОДНУ конфигурацию (tier, горизонт, норма,
   порог) по Sharpe-после-издержек. Фиксируем её письменно ДО взгляда на test.
3. **Один прогон на test.** Конфигурация считается на held-out ровно раз. Совпало —
   сигнал реален; рассыпалось — артефакт train.
4. **Форвард (режим C).** Даже прошедшая test-конфигурация подтверждается на свежих
   отчётах, накапливаемых дальше во времени, прежде чем в неё верить капиталом.
5. **Лог отброшенного.** Все протестированные конфигурации и их train/test-метрики
   логируются — чтобы multiple-testing был виден, а не спрятан.

## 8. Архитектура

Переиспуем модуль `edge/`, **дропаем LLM-путь** (`predict.py`, `prompts.py`, memory-probe
в раннере не используются). Новый под-модуль `edge/pead/` или расширение `edge/`:

- `sue.py` — расчёт SUE из `EarningsEvent` рядов (обе нормализации, point-in-time).
- `costs.py` — cost-модель по тиру (чистая функция bps → доход net).
- `portfolio.py` — квинтильная сборка + симуляция → equity curve (чистые функции).
- `study.py` — раннер развёртки на train, выбор конфигурации, один прогон на test, отчёт.
- Данные: `AlphaVantageSource.calendar` уже даёт reportedDate + EPS факт/консенсус;
  realized — существующий `realize.py` (оконный yfinance), расширить на горизонты 1-60.
- Метрики: существующий `metrics.py` + `analysis/track_record.py` (Sharpe, drawdown).

Чистая логика (SUE, costs, portfolio, metrics) — TDD на фейках; сетевые адаптеры —
graceful-degrade, как в проекте.

## 9. Гейт-решение

Edge реален и стоит торговли, только если **ВСЕ**:
1. на **held-out** test long-short net издержек тира > 0 и значимо (t-stat),
2. Sharpe оправдывает риск (и бьёт buy-and-hold SPY риск-скорректированно),
3. max drawdown в приемлемых рамках,
4. подтверждается на форвард-накоплении (режим C).

Иначе — **честный стоп**: механический PEAD не торгуем; пивот к не-торговой ценности
проекта (теперь уже заслуженный данными, а не предположением).

## 10. Вынесено за рамки (YAGNI)

- Реальная торговля и исполнение — отдельный этап, только при положительном гейте.
- LLM в любом виде — доказано бесполезен на этой задаче.
- Опционы/волатильность, intraday, плечо — другой класс, не здесь.
- Мультифакторные комбинации (PEAD × momentum × …) — только если чистый PEAD пройдёт
  гейт; иначе это перебор ради перебора.
- Оптимизация cost-чисел под результат — запрещена (это путь к фиктивному edge).
